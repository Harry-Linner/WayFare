## 一、 整体架构概述

WayFare 采用了“感知-决策-执行”的三层解耦架构。系统不仅是一个被动响应的工具，而是一个能够 24/7 异步处理资料、实时感知用户行为并主动介入学习过程的智能体（Agent）。

## 二、 核心分层详解

### 1. 沉浸式感知端 (The Sensory Frontend)

前端不仅是展示界面，更是用户行为的采集传感器。

- **双模式阅读器**：集成 `pdf.js` 与 `Milkdown`，支持 PDF 与 Markdown 的深度交互。它负责在文档原生位置动态渲染 AI 批注气泡，确保用户在不脱离原始资料的前提下获得辅助支持。
- **用户行为追踪器 (Interaction Monitor)**：实时监测用户的学习心流，包括页面停留时间、划词频率及翻页速度。这些数据作为“主动交互”的触发信号，传回调度中枢。
- **主动消息系统 (****Push** **Component)**：独立的消息分发层，用于推送日程提醒、任务截交警告以及基于当前学习重难点的即时询问。

### 2. 调度中枢层 (Tauri-Rust Orchestrator)

作为系统的控制中心，负责跨平台的资源调度与安全合规。

- **原生文件系统管理**：基于 Tauri 的 Rust 后端，严格管理用户授权文件夹的访问权限，确保数据安全。
- **异步任务调度 (Job Queue)**：管理后台静默解析任务。当用户放入海量资料时，中枢会根据当前阅读重心动态分配计算优先级，确保“当前必看”资料被最快处理。
- **事件触发引擎 (Trigger Engine)**：匹配行为数据与预设逻辑。例如，当“停留时间 > 阈值”且“内容难度 > 均值”时，主动唤起内核进行干预。

### 3. 智能管家内核 (The Cognitive Sidecar Kernel)

这是 WayFare 的灵魂，由高性能 Go/Rust 驱动，负责所有重计算逻辑。

- **认知心理学引擎 (Pedagogical Engine)**：
    - **费曼转化器**：利用 LLM 将晦涩的学术定义转化为类比和通俗逻辑。
    - **认知支架模块**：为每个知识点自动构建学习路径，生成由浅入深的引导式问题。
- **个性化记忆系统 (Memory & Profile)**：
    - **长期记忆库**：记录用户的历史错题、理解偏好及已知知识图谱。
    - **动态用户画像**：随着交互深入，自动调整输出风格（如：从“严谨学术型”转向“直觉引导型”）。
- **增强理解模块 (Retrieval & Search)**：
    - **本地 RAG 引擎**：利用本地向量数据库实现对所有私有资料的毫秒级检索。
    - **联网增强 Agent**：当本地资料无法支撑深度解释时，自动检索网络学术资源进行补充批注。

## 三、 核心工作流：从“静态资料”到“动态陪伴”

1. **静默解析 (异步模式)**： 用户将资料拖入文件夹。`File Watcher` 立即唤醒内核，在后台完成重点识别、考频分析（对比历年真题）并生成 `JSON` 批注文件。
2. **主动介入 (实时模式)**： 当用户阅读到某一真题考点但长时间停滞时，前端感知信号传回中枢，触发 `Pedagogical Engine` 发送主动消息：“这是一个高频考点，但在之前的模拟中你曾做错，需要我为你重新拆解逻辑吗？”
3. **认知沉淀 (闭环模式)**： 所有的交互记录、用户对批注的反馈、AI 的讲解细节，都会回流至 `SQLite` 数据库。这种认知痕迹随着时间积累，形成一个“越用越懂你”的学习生态。

## 四、 技术栈总结

- **前端**：React / Vue 3 + Tailwind CSS + pdf.js + Milkdown
- **中枢**：Tauri (Rust)
- **内核**：Go (Sidecar 模式)
- **存储**：SQLite (加密本地存储) + ONNX (本地向量计算)
- **模型逻辑**：深度微调的教育专用模型接口 + 自研 Prompt 工作流
