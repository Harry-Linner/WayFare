## 🏛️ 整体架构设计：分层微服务

我们将系统分为四个主要逻辑层。这种设计利用了 Go 的高并发能力和 C++/Python 在计算与 AI 逻辑上的优势。

### 1. 前端接入层 (Frontend & Backend-for-Frontend)

- **技术栈：** Astro + TypeScript + Tailwind CSS
- **职责：** * **Astro** 负责高性能的静态页面渲染（内容展示、文档）。
  - **TypeScript** 用于编写复杂的客户端交互逻辑。
  - 利用 Astro 的 SSR 能力处理用户认证和基础 API 路由。

### 2. 核心调度层 (Orchestration Layer)

- **技术栈：** Go
- **职责：** * **API Gateway：** 处理 WebSocket 连接（实时学习反馈）和 REST 请求。
  - **权限校验：** 实现你提到的“谨慎权限”控制，通过 JWT 和 RBAC 限制用户操作。
  - **任务分发：** 将计算密集型任务或 AI 请求分发到后端的 Python 或 C++ 模块。

### 3. 执行与计算层 (Execution Layer)
这是应用的核心，针对“学习类”需求进行差异化设计：

- **Python (AI & 逻辑处理)：** * 用于集成 LLM（如处理学习建议、代码解释）。
  - 使用 Fast API 或 gRPC 与 Go 通信。
- **C/C++ (底层安全沙箱/高性能计算)：**
  - 如果应用涉及代码在线执行，使用 **C/C++** 编写极简的 WebAssembly 运行时或 Linux Namespaces 容器沙箱，确保用户代码运行在受限环境（这符合你“谨慎权限”的要求）。

### 4. 数据持久化层 (Data Layer)

- **技术栈：** SQL (PostgreSQL/MySQL) + Redis
- **职责：** * **SQL：** 存储用户信息、学习进度、结构化课程内容。
  - **Redis：** 缓存活跃的 Session 和临时状态。

## 🛠️ 关键模块实现思路

### 权限与安全（针对“更小更谨慎”）
为了实现权限控制，我们可以在 Go 调度层引入 **Policy Engine**：

- **最小权限原则：** 每个学习任务只分配必要的资源。
- **指令拦截：** 在 Python/C++ 执行层前置一层过滤，防止敏感系统调用。

### 跨语言通信
由于技术栈涵盖了 Go、Python 和 C++，建议使用 **gRPC** 作为内部通信协议：

- **定义 .proto 文件：** 统一数据交换格式。
- **高性能：** 远比 HTTP JSON 序列化更快，适合实时学习互动。

###  成员 A：全栈与交互架构师（Frontend & API Gateway）

**核心职责：** 把控用户感知的“第一线”和系统的“进出口”。

- **前端开发：** 使用 **Astro + TS** 搭建高性能的学习界面。既然是学习类应用，响应式布局和加载速度（Astro 的优势）至关重要。
- **网关接入：** 使用 **Go** 编写 API Gateway。处理 WebSocket 实时通信、用户认证（JWT）以及基础的路由转发。
- **状态管理：** 负责定义前端与后端交换的数据结构（TS 类型定义）。
- **技能侧重：** Astro / TS / Go / CSS。

### 👤 成员 B：业务逻辑与 AI 工程师（Logic & AI Orchestration）
**核心职责：** 处理“大脑”部分的逻辑，让应用“聪明”起来。

- **AI 服务：** 使用 **Python** 编写核心逻辑，对接 LLM（如处理学习建议、题目解析）。
- **业务层 API：** 负责核心业务逻辑（如学习进度算法、课程管理）。
- **数据库设计：** 负责 **SQL** 模式设计，管理用户数据、课程内容和日志。
- **与 A 配合：** 确保 Go 网关能顺畅调用 Python 编写的服务。
- **技能侧重：** Python / SQL / Go。

### 👤 成员 C：底层安全与执行工程师（System & Sandbox）
**核心职责：** 解决“权限更小、更谨慎”的关键技术指标。

- **安全沙箱：** 使用 **C/C++** 开发受限的执行环境。如果学习应用涉及运行代码，由你负责拦截敏感系统调用，确保安全。
- **底层优化：** 编写高性能的 C++ 模块或 WebAssembly 组件，用于处理复杂的计算任务。
- **DevOps/部署：** 负责容器化（Docker）和系统级安全配置，确保整个应用架构的隔离性。
- **技能侧重：** C / C++ / Linux 系统编程。

### 🔄 团队协作流程建议
为了让三个人高效同步，建议采用以下流程：

1. **定义接口 (Contract First)：** 三个人先坐在一起，用 **SQL** 定义好核心表结构，用 **Protobuf** 定义好 Go、Python 和 C++ 之间的通信接口。
2. **并行开发：**
  - **A** 模拟假数据开发前端界面。
  - **B** 在本地跑通 AI 提示词（Prompt）逻辑。
  - **C** 预研安全沙箱的权限隔离方案。
